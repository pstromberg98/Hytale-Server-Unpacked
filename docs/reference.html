Components

Components are pure data attached to entities. Access via store.getComponent(ref, ComponentClass.getComponentType()).

Entity Classes

Entity classes form the inheritance hierarchy. Most game objects are pure-component entities, but these classes provide base functionality.

ClassExtendsPackageDescription
Entity—...core.entityAbstract base. Has TransformComponent, network ID, UUID, display name.
LivingEntityEntity...core.entityAbstract. Adds Inventory, fall damage, stat modifiers, equipment.
PlayerLivingEntity...entity.entitiesPlayable character. Input, camera, HUD, permissions, game mode.
BlockEntityEntity...entity.entitiesBlock-based entities: chests, furnaces, doors, interactive blocks.
Note

Most game objects (NPCs, projectiles, items, props) are not Entity subclasses. They are pure component entities created via Holder + Store.addEntity(). NPCs use the NPCEntity component, not an Entity subclass.

Systems

Systems contain the logic that processes entities with specific component combinations each tick. Components are data; systems are behavior.

Tip

Systems are where behavior lives. If components are the "nouns", systems are the "verbs". When you add a Teleport component to an entity, a system picks it up and performs the teleport. When you add DespawnComponent, the DespawnSystem handles the countdown and removal.
